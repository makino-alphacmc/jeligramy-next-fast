# 共通レイアウト・コンポーネント定義

全画面で共通して使うパーツの仕様。SP/PC で寸法のみ変更し、見た目は統一する。

---

## 1. ヘッダー (Header)

### 役割
- サイト名（ロゴ）の表示
- ユーザーアイコン（プロフィール・メニュー入口）の配置
- **コンテンツエリア上のみ**に表示（サイドメニューの右側）。全幅ではない。
- 全画面で固定表示（sticky）を推奨

### 仕様

| 項目 | SP | PC |
|------|-----|-----|
| 高さ | 56px | 56px |
| 背景 | `--card`（`#2d2e3a`） |
| ボーダー | 0.5px `--border`、角丸 `--radius-lg`（10px） |
| 左右パディング | 24px | 24px（コンテンツエリア内） |

### 配置
- **左**: サイト名「jeligramy」  
  - フォント: system-ui, 18px, font-weight 600, 色 `#fafafa`
- **右**: ユーザーアイコン（アバター or アイコン）  
  - 幅 40px、高さ 32px、角丸 8px  
  - 背景 `--primary`（`#e4e4e7`）、アイコン色 `--primary-foreground`（`#2d2e3a`）  
  - クリックでプロフィール・設定・ログアウト等のメニューを想定

### 使用画面
- トップ、新規投稿、その他全画面

---

## 2. サイドメニュー (Side Menu)

### 役割
- 主要ナビゲーション（ホーム、新規投稿、プロフィールなど）の配置
- PC: 画面左に常時表示。SP: ハンバーガー等で開くドロワー、または下部タブバー
- **下端はメインコンテンツ・フッターと揃える**（フッター手前に余白を取る）

### メニュー項目（上から）

| 表示名 | 役割 |
|--------|------|
| Home | 投稿一覧（トップ）へ |
| New post | 新規投稿画面へ |
| Profile | 自分のプロフィール・投稿履歴へ |
| Drafts | 下書き一覧へ（任意） |
| Settings | アカウント・表示設定などへ |
| Logout | ログアウト（**区切り線の下**に配置） |

### 仕様

| 項目 | SP | PC |
|------|-----|-----|
| 幅 | ドロワー時 280px / タブバー時 全幅 | 200px（小） / 300px（大）※画面幅に応じて |
| 背景 | `--sidebar`（= `--card` `#2d2e3a`） | 同上 |
| ボーダー | 0.5px `--border`、角丸 10px | 同上 |
| 項目の高さ | 48px | 48px |
| 項目の左右パディング | 16px | 24px |
| フォント | 14px, font-weight 500 | 14px, font-weight 500 |

### 配置
- 縦並びリスト。ラベル（任意でアイコン + ラベル、SP ではアイコンのみ可）
- 1行目は区切り線（セパレータ）。ログアウトの上にも区切り線を入れる。
- アクティブ: 文字 `--foreground`（`#fafafa`）
- 非アクティブ: 色 `--muted-foreground`（`#a1a1aa`）, hover `--foreground`
- ログアウト: 区切り線の下、色 `--muted-foreground`

### 使用画面
- 全画面。PC では常に左側、SP ではドロワー/タブで表示

---

## 3. パンくずリスト (Breadcrumb)

### 役割
- 現在位置の階層表示（例: Home / Posts）
- クリックで上位階層へ遷移

### 仕様

| 項目 | SP | PC |
|------|-----|-----|
| 高さ | 36px | 36px |
| 背景 | なし（コンテンツエリア内） |
| 左右パディング | 24px | 24px（コンテンツエリア内） |
| フォント | 12px, 色 セパレータ・リンク `--muted-foreground`（`#a1a1aa`）、セパレータ `/` は `#71717a` |

### 配置
- セパレータ: `/` 色 `#71717a`
- リンク: 色 `--muted-foreground`, hover `--foreground`
- 現在ページ: 色 `--foreground`（`#fafafa`）, リンクなし

### 使用画面
- 2階層以上ある画面（例: トップの Posts、投稿詳細、編集画面）

---

## 4. フッター (Footer)

### 役割
- コピーライト、利用規約・プライバシーリンクなど
- **画面全幅**でコンテンツ最下部に配置

### 仕様

| 項目 | SP | PC |
|------|-----|-----|
| 高さ | 36px | 36px |
| 背景 | **黒** `#000` |
| ボーダー | なし |
| テキスト | **中央揃え**、左右パディングはコンテンツ幅に合わせて可 |

### 配置
- **中央揃え**
- 1行目: コピーライト（例: c 2026 jeligramy）、11px、色 `#71717a`
- 2行目: Terms | Privacy 等のリンク、11px、リンク色 `#a1a1aa`、区切り `|` は `#71717a`
- リンク: hover 時 `--foreground` を想定

### 使用画面
- 全画面（メインコンテンツ・サイドメニュー下端の下）

---

## 5. メインコンテンツエリア (Main Content Area)

### 役割
- 各画面の本文・一覧・フォーム等を表示する領域
- **スクロール可能**であることを示す UI（長いコンテンツ時）

### 仕様

| 項目 | SP | PC |
|------|-----|-----|
| 位置 | ヘッダー・パンくずの下、フッターの上 | 同上（サイドメニュー・ヘッダーの右） |
| 背景 | `--card`（`#2d2e3a`） | 同上 |
| ボーダー | 0.5px `--border`、角丸 10px | 同上 |
| スクロール表現 | 点線枠（6 4 dash）、右端にスクロールバー、下端にフェード | 同上 |

### スクロール UI（オプション）
- **点線枠**: `stroke-dasharray: 6 4`、色 `rgba(255,255,255,0.06)`
- **スクロールバー**: トラック幅 6px、角丸 3px、色 `rgba(255,255,255,0.06)`。サムは幅 4px、角丸 2px、色 `rgba(255,255,255,0.25)`
- **下端フェード**: コンテンツ下端にグラデーション（70% 透明 → 100% 背景色 `#1a1b24`）で「続きあり」を表現

### 配置
- サイドメニュー・ヘッダーと**下端を揃える**（フッター手前に同一の余白）
- 左右パディングは画面により可変（例: トップのカードグリッド時は 32px 等）

### 使用画面
- 全画面

## 6. 投稿カード（トップ一覧用）

トップ画面の投稿一覧で使用するカードの共通仕様。他画面でカードを使う場合も参考にする。

### 構成（上から）
1. **画像エリア**（カード上部）  
   - アスペクト・高さは実装で調整。角丸 10px（上のみ、または全体で 10px）。
2. **投稿者エリア**（画像エリア内・左上）  
   - プロフィール写真: 円形、径約 28px、背景 `#52525b`、枠線 `rgba(255,255,255,0.3)`  
   - その横に**ユーザー名**（例: username）、11px、font-weight 500、色 `rgba(255,255,255,0.95)`
3. **3点メニュー**（画像エリア内・右上）  
   - 「・・・」の縦または横 3 点、色 `#a1a1aa`。クリックでメニュー（編集・削除等）を想定。
4. **タイトル・説明**（画像下）  
   - プレースホルダまたは 1〜2 行テキスト。背景 `--card`、角丸 6px のブロックまたはテキスト。色・opacity は `--muted` 系で調整。
5. **アクション行**（カード最下部）  
   - **いいね**: アイコン ♡（未いいね） / ❤（いいね済）、横に**件数**（例: 12）。色 `#a1a1aa`、10px。  
   - **コメント**: 吹き出しアイコン、横に**件数**（例: 3）。色・サイズはいいねと統一。

### 仕様
- カード全体: 背景 `--card`、ボーダー 0.5px `--border`、角丸 10px、シャドウ（任意、例: drop-shadow 0 2px 4px rgba(0,0,0,0.08)）
- アイコンと数字のサイズは**統一**（例: アイコン高さ約 10px、数字 10px）
- 左右パディングはコンテンツエリア内で**同一**（例: 32px）。スクロールバーと被らないよう余白を取る。

### 使用画面
- トップ（投稿一覧）

## レイアウト構成

### PC（横方向）

```
[ サイドメニュー ] | [ ヘッダー（コンテンツエリア上） ]
                   | [ パンくず ]  ※必要な画面のみ
                   | [ メインコンテンツ（スクロール可能） ]
─────────────────────────────────────────────────────────
[ フッター（画面全幅・黒背景・中央揃え） ]
```

- サイドメニューとメインコンテンツの**下端は揃える**（フッター手前に余白）。
- ヘッダーはコンテンツエリア内のみ（全幅ではない）。右端はユーザーアイコン。

### SP（縦方向）

```
[ ヘッダー ]
[ パンくず ]      ※必要な画面のみ
[ メインコンテンツ ]
[ フッター（全幅・黒・中央揃え） ]

※ サイドメニューはドロワー or 下部タブで表示
```

## カラー・トークン（shadcn / 本プロジェクト dark テーマ）

`frontend/src/app/globals.css` の `.dark` と同期。実装では CSS 変数を参照すること。

| 用途 | CSS変数 | 値（参考・hex） |
|------|---------|-----------------|
| 背景 | `--background` | `#1a1b24` |
| 本文 | `--foreground` | `#fafafa` |
| カード・サイドバー | `--card` / `--sidebar` | `#2d2e3a` |
| カード文字 | `--card-foreground` | `#fafafa` |
| ミュート背景 | `--muted` | `#3f3f46` |
| サブテキスト | `--muted-foreground` | `#a1a1aa` |
| ボーダー | `--border` | `rgba(255,255,255,0.1)` |
| プライマリ（ボタン等） | `--primary` | `#e4e4e7` |
| プライマリ文字 | `--primary-foreground` | `#2d2e3a` |
| 危険（削除等） | `--destructive` | `#dc2626` |

## 角丸（shadcn --radius）

| 用途 | トークン | 値 |
|------|----------|-----|
| 小（input等） | `--radius-sm` | 6px |
| 中（ボタン等） | `--radius-md` | 8px |
| 大（カード等） | `--radius-lg` | 10px |

※ 本プロジェクトの `--radius: 0.625rem`（10px）基準。
